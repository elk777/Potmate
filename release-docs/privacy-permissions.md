# 权限与隐私说明

Potmate「小盆友」的桌面能力以本地体验为主，不需要账号，也不上传记录内容。本文面向普通用户解释每项权限为什么存在，以及它不会做什么。

## 本地数据

- 每日记录、植物状态、图鉴和设置存储在本机。
- 桌面端优先使用 Tauri Store。
- 浏览器预览模式使用 `localStorage`。
- 备份文件由用户手动导出和导入。
- 当前版本没有云同步和服务器上传。

## 通知权限

用途：在用户开启每日提醒后，发送当天未记录的小提醒。

边界：

- 用户关闭每日提醒后不主动发送通知。
- 通知内容不包含用户记录原文。

## 开机自启

用途：用户开启后，让桌宠随系统启动。

边界：

- 仅在用户打开设置开关后启用。
- 用户关闭开关后调用系统能力取消自启。

## 键盘和鼠标活动

用途：作为辅助养分信号，估算用户是否仍在电脑前活动。

边界：

- 只记录点击/按键事件类型和次数。
- 不记录按下了哪个具体键。
- 不记录鼠标坐标。
- 不读取输入内容。
- 不上传任何活动数据。
- 权限不足时自动降级为应用窗口内活动监听。

该能力是可选功能。不开启时，主线每日记录养花仍然可用。

## 文件读写

用途：通过系统文件选择器导入和导出 Potmate JSON 备份。

边界：

- 不主动扫描用户目录。
- 不读取非用户选择的文件。
- 无效备份文件不会覆盖当前花园数据。

## 画布穿透和置顶窗口

用途：让桌宠像桌面小物件一样停留在屏幕上，并在开启穿透后允许点击空白区域下方的内容。

边界：

- 画布穿透只改变窗口鼠标事件行为。
- 不读取桌面下方应用内容。
- Linux 下表现可能受桌面环境和窗口管理器影响。

## macOS 权限提示

全局键鼠监听可能触发辅助功能或输入监控权限。拒绝授权不会影响主线记录养花，只会让副方案活动养分退回窗口内监听。
